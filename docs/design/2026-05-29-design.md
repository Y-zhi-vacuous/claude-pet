# 桌面比熊小狗桌宠 — 设计文档

> 把 Claude Code 变成 Windows 桌面动画小狗伴侣

---

## 1. 项目定位

一只比熊风格的 2D 手绘小狗，浮在 Windows 桌面上，作为 Claude Code 的化身。

- **身份**：Claude 的桌面皮肤，不是独立宠物
- **形态**：约 140×140px 精灵动画
- **核心能力**：能聊天、能操控 Claude、能回答问题、能显示 Claude 工作状态

---

## 2. 整体架构

```
小狗 WPF 壳（主进程）
    │
    ├── 只读通道：FileSystemWatcher → transcript JSONL → HUD 状态
    │
    └── 读写通道：Process → claude -p "..." --continue → 代码操作
```

**设计原则**：
- 小狗是主控端，Claude Code CLI 是子进程
- 任何交互最终变成 `claude -p` 调用
- 不修改 Claude Code 内部逻辑，只套桌面外衣
- 工程文件放 D:\dev\claude-pet\，不污染 C 盘

---

## 3. 组件拆分

| 组件 | 职责 | 一句话 |
|------|------|--------|
| PetWindow | 透明窗口 + 小狗渲染 | 无边框、穿透点击、总在最前、可拖移 |
| SpriteEngine | 精灵动画 | 帧播放、状态切换、实现 IAnimationPlayer |
| BubbleOverlay | 聊天气泡 | 对话历史、输入框、语音文字展示 |
| ClaudeBridge | CLI 管理 | 进程启停、-p 发指令、解析 transcript |
| VoiceEngine | 语音 | Porcupine 唤醒 → VAD → STT → TTS |
| DropTarget | 拖拽 | 文件→窗口→prompt 构造 |
| TrayIcon | 托盘 | 右键菜单：退出/暂停/设置/切换会话 |
| ConfigStore | 配置 | 热加载 config.json |

中心组件 PetWindow 通过 Mediator 模式协调所有子组件。

## 4. 精灵动画系统

### 状态机

```
        ┌──────────────────────┐
        │       睡觉 (idle 5min) │ ← 长时间无操作
        └──────────┬───────────┘
                   │ 鼠标移动/唤醒词
                   ▼
    ┌──────┐   ┌──────┐   ┌──────┐
    │ 待机  │──→│ 走动  │──→│ 玩耍  │
    │呼吸眨眼│  │ 4方向  │   │追尾巴翻滚│
    └──┬───┘   └──────┘   └──────┘
       │
       │ Claude 状态驱动
       ▼
    ┌──────┐   ┌──────┐   ┌──────┐
    │ 思考 💭│   │ 说话 📢│   │ 开心 🎉│
    │歪头踱步│   │叫摇尾巴│   │转圈跳跃│
    └──────┘   └──────┘   └──────┘
```

### 技术实现
- 精灵帧：每个动画一个 PNG 序列帧文件夹
- 渲染：WriteableBitmap + 后台定时器，12fps
- 接口：IAnimationPlayer 抽象，初始 SpriteSheetPlayer，预留 Spine/Live2D 扩展

### IAnimationPlayer 接口

```csharp
public interface IAnimationPlayer
{
    void Play(string state);
    void Stop();
    void SetPosition(double x, double y);
    (int Width, int Height) GetSize();
    event Action<string> AnimationEnded;
}
```

切换实现只需改一行 new，接口不变。

## 5. 语音引擎

### 流水线

```
Porcupine 唤醒词 → WebRTC VAD (静音检测) → 云端 STT → 文本送 ClaudeBridge
                                                              ↓
NAudio 播放 ← 云端 TTS ← Claude 回复文本
```

### 选型

| 组件 | 选型 | 模式 |
|------|------|------|
| 唤醒词 | Porcupine C# SDK | 本地离线 (~2MB) |
| VAD | WebRTC VAD C# 移植版 | 本地离线 (~100KB) |
| STT | Azure Speech / API | 云端 |
| TTS | Azure Speech / Edge TTS | 云端 |

### 状态机

```
LISTENING → WAKED_UP → RECORDING → PROCESSING → SPEAKING → LISTENING
                ↑ 小狗"思考"动画       ↑ 小狗"说话"动画
```

## 6. ClaudeBridge 通信

### 指令模式
- 使用 `claude -p "指令" --continue`
- 每次发指令启动短期进程，结束退出
- `--continue` 确保在同一会话上下文中

### Transcript 监控
- FileSystemWatcher 监听 `~/.claude/transcripts/<latest>.jsonl`
- 增量读取，只解析新增行
- 提取 model、context、tool_use、tool_result、Task、TodoWrite
- 聚合成 StateSnapshot 推送 PetWindow

### 心跳
- 每 5 秒检查 Claude 进程存活
- 挂了自动重启，--continue 恢复
- 小狗"打哈欠"表示等待

### StateSnapshot

```csharp
public class StateSnapshot
{
    public string Model { get; set; }
    public double ContextPercent { get; set; }
    public int InputTokens { get; set; }
    public int MaxTokens { get; set; }
    public List<ToolActivity> ActiveTools { get; set; }
    public List<AgentInfo> RunningAgents { get; set; }
    public List<TodoItem> Todos { get; set; }
    public string Status { get; set; } // "idle" | "working" | "error"
}
```

## 7. UI 交互层

### PetWindow
- `WindowStyle=None` + `AllowsTransparency=True`
- `Topmost=true`，浮于桌面
- 小狗区域不穿透，边角空白区域穿透

### 交互行为

| 操作 | 触发方式 | 行为 |
|------|---------|------|
| 点击小狗 | 左键单击 | 弹出/收起聊天 Bubble |
| 拖拽小狗 | 按住左键拖动 | 移到新位置，写入 config |
| 拖文件到小狗 | 资源管理器拖入 | 小狗"叼住"动画，文件路径拼进 prompt |
| 右键菜单 | 右键点击 | 暂停动画/切换会话/设置/退出 |
| 唤醒词 | "嘿小狗" | 小狗醒来，准备听指令 |
| 语音输入 | 唤醒后说话 | STT → 文本 → Claude |
| 双击小狗 | 左键双击 | 玩耍动画（纯卖萌） |
| Claude 工作中 | 自动 | 思考动画 + 迷你转圈 |
| Claude 完成 | 自动 | 开心动画 + 气泡摘要 |

## 9. 启动流程

```
1. 托盘图标出现在通知区域
2. PetWindow 在屏幕右下角弹出，播放"待机"动画
3. ClaudeBridge 检查 claude 是否可用
4. VoiceEngine 后台启动唤醒词监听
5. 小狗眨眨眼，开始等你
```

## 10. 分阶段实施

### P1 — 骨架（先跑起来）
- WPF 透明窗口 + 彩色方块占位
- 拖拽移动 + 托盘
- SpriteSheetPlayer 基础版（方块换色模拟动画）

### P2 — 大脑（接入 Claude）
- ClaudeBridge（ProcessManager + TranscriptWatcher）
- `-p --continue` 指令发送
- HUD 迷你状态条

### P3 — 外观（精灵替换）
- 比熊精灵帧制作/采购
- 全部动画状态接入
- 聊天气泡 UI

### P4 — 声音（语音引擎）
- Porcupine 唤醒词
- STT + TTS 接入
- 语音交互闭环

### P5 — 打磨
- 拖拽文件
- 动画过渡
- 配置面板
- 错误恢复
